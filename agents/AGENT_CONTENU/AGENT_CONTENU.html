Livrable #5 — Contenu pédagogique

Module E-Learning

Module de formation IA structuré selon le framework OPAD — nourri par les sources qualifiées, le Parcours Blended et la Maquette Pédagogique du projet.

Sources (priorité décroissante)
1
Brief E-Learning — Parcours Blended (#4)
Identifie quel module produire, le moment (Explorer / Enrichir) et la fonction pédagogique
2
Maquette Pédagogique (#2)
Objectifs Bloom, public cible, prérequis, durée — aligne le module sur le cadre global
3
Sources qualifiées
Transcripts YouTube / Podcast / Google Drive — max 5, tronquées à 8 000 chars chacune
4
Paramètres manuels
Thème, persona, stade, durée, département — fallback si aucun contexte projet disponible
Framework OPAD — 4 sections obligatoires dans cet ordre
O
Objectif
1 objectif Bloom adapté au stade. Format : "À l'issue de ce module, [public] sera capable de [verbe] [compétence] dans [contexte]".
Si Maquette disponible → aligner obligatoirement sur ses objectifs.
P
Pratique
Exercice guidé pas-à-pas en autonomie. 30–50% de la durée totale. Si Explorer → découverte. Si Enrichir → ancrage post-salle.
Utiliser exemples tirés des sources qualifiées.
A
Ancrage
Cas concret métier issu du secteur et du public cible. Ancré dans une situation terrain réelle (deskless workers, smartphone).
Extraire un extrait du transcript si disponible.
D
Déploiement
1 action concrète terrain dans les 24–48h suivant le module. Format : verbe + objet + contexte + résultat attendu.
Si Enrichir → prolonge ce qui a été fait en salle.

Personnalisation workflow
Évaluation finale (obligatoire)

3 questions max — QCM ou Vrai/Faux — avec feedback immédiat par réponse.

Si Questionnaire de positionnement disponible : aligner les questions pour mesurer la progression entre avant et après formation (ROI pédagogique Qualiopi).
